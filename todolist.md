# FitVision 用户体验版 ToDoList（学生项目）

> 目标：做一个“好用、好玩、好展示”的 AI 健身项目，不追求复杂工程化。
> 核心原则：优先做用户能直接感知的体验。

---

## 一、先解决用户视角的 5 个痛点

- [ ] 用户不知道“该怎么开始”：缺少新手引导与训练前准备页
- [ ] 用户不知道“动作是否标准”：缺少清晰示范与分步骤提示
- [ ] 训练过程容易枯燥：语音反馈内容单一，缺少节奏感
- [ ] 用户看不到进步：缺少直观的成长反馈与奖励感
- [ ] 训练结束后缺少行动建议：报告有结论，但缺少“下一步做什么”

---

## 二、必做（优先级最高，1-2 周内）

### 1. 训练前：加标准动作示范页（强烈建议先做）

- [ ] 每个动作增加“标准演示卡片”：GIF/短视频 + 关键点图
- [ ] 增加“常见错误 3 条”与“安全提醒 2 条”
- [ ] 增加“开始前 10 秒准备倒计时”

完成标准：
- [ ] 用户点击开始训练前，能清楚看到正确动作和注意事项

---

### 2. 训练中：语音反馈更丰富（你提到的重点）

- [ ] 语音从“计数”升级为“计数 + 鼓励 + 纠错”三类
- [ ] 增加不同语气包：鼓励型、严格型、活力型
- [ ] 增加节奏语音：
  - 下蹲阶段：慢一点/稳住
  - 起身阶段：发力/呼气
- [ ] 增加防打扰机制：3-5 秒内不重复同一句

完成标准：
- [ ] 用户训练 1 分钟内至少听到 3 类不同提示，且不觉得“吵”

---

### 3. 训练中：动作质量可视化

- [ ] 增加“动作标准度进度条”（实时变化）
- [ ] 增加“本次最常见错误”小标签（如膝盖内扣）
- [ ] 增加“连续标准动作 streak（连击）”显示

完成标准：
- [ ] 用户可以一眼知道“做得好不好、哪里不好”

---

### 4. 训练后：报告更像“教练复盘”

- [ ] 报告页新增 3 块：
  - 本次亮点
  - 需要改进
  - 下一次建议
- [ ] 增加“一键继续下一练”按钮（不要让用户回首页再找）
- [ ] 增加“推荐下一个动作（1-2 个）”

完成标准：
- [ ] 用户看完报告后能马上知道下一步做什么

---

### 5. 首页：做出“成长感”

- [ ] 增加“本周已训练 X 天”
- [ ] 增加“连续打卡天数”
- [ ] 增加“本周进步动作”
- [ ] 加一个简单成就系统（如：完成首训、连续3天）

完成标准：
- [ ] 用户每次打开首页都能看到正反馈

---

## 三、可选加分（比赛展示很加分）

### A. AI 教练人格

- [ ] 用户可选择教练风格：温柔/热血/严格
- [ ] 同样的纠错内容，用不同话术表达

### B. AI 训练前问诊（轻量）

- [ ] 训练前 3 个问题：今天状态、疲劳程度、想练部位
- [ ] 自动给出“今日轻量/标准/强化”方案

### C. AI 训练后一句话总结卡

- [ ] 自动生成一句可分享文案（朋友圈/群聊）
- [ ] 示例：今天深蹲稳定性提升 12%，继续保持

### D. 游戏化互动

- [ ] 增加“挑战模式”：1 分钟挑战计数
- [ ] 增加“好友 PK（本地假数据也可）”

---

## 四、网页美观化专项（去“AI味过重”，提升观感）

> 目标：页面看起来像“运动产品”，而不是“AI 演示面板”。

### 4.1 视觉方向统一（先定风格再改页面）

- [ ] 从“科幻 AI 风”改为“运动活力风”
- [ ] 主色建议：活力橙/青柠绿 + 中性色（减少大面积紫蓝）
- [ ] 每页保留 1 个重点色，不要全页面彩虹色
- [ ] 统一圆角、阴影、间距规范（8/12/16/24）

完成标准：
- [ ] 首页、训练页、报告页风格一致，第一眼有“健身产品感”

---

### 4.2 导航与信息层级优化（让页面更清爽）

- [ ] 顶部导航只保留核心入口（首页/训练/动作/数据/我的）
- [ ] 次要功能放入“更多”菜单，减少认知负担
- [ ] 页面标题区统一：标题 + 副标题 + 主按钮
- [ ] 同屏卡片数量控制，避免“信息墙”

完成标准：
- [ ] 新用户 5 秒内能找到“开始训练”入口

---

### 4.3 训练前中后三段式视觉重做

#### 训练前（准备感）
- [ ] 加“今日训练卡片”：目标部位、预计时长、建议强度
- [ ] 动作示范卡片做大图优先，文字精简

#### 训练中（专注感）
- [ ] 页面只保留核心元素：摄像头、计数、语音提示、暂停/结束
- [ ] 把次要信息折叠到抽屉面板

#### 训练后（成就感）
- [ ] 报告页突出“分数 + 亮点 + 下一步”
- [ ] 增加“继续下一练”主按钮

完成标准：
- [ ] 用户完成一次训练流程时，不会觉得页面杂乱

---

### 4.4 字体与文案去“技术腔”

- [ ] 文案从“算法/诊断/分析”改成“教练式表达”
- [ ] 减少“AI”字样重复出现（保留必要亮点即可）
- [ ] 状态提示更口语：
  - 原：姿态分析中
  - 新：正在帮你看动作，再坚持一下

完成标准：
- [ ] 用户反馈“像一个健身助手”，而不是“技术后台”

---

### 4.5 轻动画与反馈（提升高级感）

- [ ] 页面进入时做轻量渐入（150-250ms）
- [ ] 关键操作加反馈动画（开始训练、完成训练、达成目标）
- [ ] 按钮状态明确：默认/悬停/点击/禁用

完成标准：
- [ ] 交互更顺滑，不出现明显“突变”

---

### 4.6 文件级实现清单（可直接开工）

- [ ] `frontend/src/style.css`：重做全局色板、字体、阴影、间距变量
- [ ] `frontend/src/layouts/MainLayout.vue`：精简菜单项、优化导航层级
- [ ] `frontend/src/views/HomeView.vue`：重做首页首屏（今日训练 + 快速开始）
- [ ] `frontend/src/views/TrainingView.vue`：改为“专注模式”布局（主画面优先）
- [ ] `frontend/src/views/TrainingReportView.vue`：增强成就感与下一步引导

完成标准：
- [ ] 五个核心页面视觉一致，且移动端可正常显示

---

### 4.7 美观化验收（简单有效）

- [ ] 找 3 位同学做 A/B 主观评分（旧版 vs 新版）
- [ ] 评分维度：清爽度、易用性、运动感、继续使用意愿（1-5分）
- [ ] 平均分提升 1 分以上视为有效

---

## 五、最小 AI 展示点（够比赛答辩用）

- [ ] AI 实时纠错（训练中）
- [ ] AI 个性化建议（训练后）
- [ ] AI 今日计划推荐（训练前）

答辩时重点讲这三句话：
- [ ] 训练前：AI 帮用户定计划
- [ ] 训练中：AI 帮用户纠动作
- [ ] 训练后：AI 帮用户做复盘

---

## 六、两周简化排期（学生友好）

### 第 1 周（把体验做顺）

- [ ] 完成训练前动作示范页
- [ ] 完成训练中语音增强
- [ ] 完成实时标准度进度条
- [ ] 完成网页美观化基础改造（全局色板 + 导航 + 首页首屏）

### 第 2 周（把体验做亮）

- [ ] 完成训练后复盘升级
- [ ] 完成首页成长反馈 + 成就系统
- [ ] 完成 1-2 个 AI 加分项（建议选 A + B）
- [ ] 完成训练页/报告页视觉重做 + 轻动画

---

## 七、验收方式（不复杂但实用）

- [ ] 找 3-5 个同学试用
- [ ] 每人完成一次完整流程（训练前 -> 训练中 -> 训练后）
- [ ] 收集 3 个问题：
  - 哪里看不懂？
  - 哪里不好用？
  - 哪个功能最喜欢？
- [ ] 根据反馈做一轮快速修改

---

## 八、里程碑（简化版）

- M1：流程顺（能完整练完一次）
- M2：反馈强（语音、示范、复盘都可感知）
- M3：展示亮（有 AI 特色且好讲解）
